## 4. Demo Flow Trace (PRD §9 — 5 Steps)

This section traces the exact handler chain for each of the 5 demo steps through the source code.

### Step 1 — Réceptionner un lot de viande de bœuf

**Route:** `/matieres-premieres`  
**Trigger:** "+ Réceptionner un lot" button → `setDialogOpen(true)` in `app/matieres-premieres/page.tsx`  
**Component:** `components/matieres/reception-dialog.tsx`  
**Handler chain:**
1. User fills 9-field form (Type: boeuf, Nom, Fournisseur, N° lot, Quantité, Date réception, DLC, Température ±Certificat)
2. Zod schema validates: all 8 required fields present + DLC > dateReception cross-field refine
3. `onSubmit()` → `crypto.randomUUID()` → `quantiteRestante = quantiteRecue` → `useTraceabilityStore.getState().addRawMaterial(newRm)` → `toast.success()` → `onOpenChange(false)`
4. `app/matieres-premieres/page.tsx` re-renders via Zustand selector; new lot visible in table

**Status: WIRED** — all 4 links confirmed at lines 108–109 of reception-dialog.tsx

### Step 2 — Lancer un ordre de fabrication (4 broches)

**Route:** `/production`  
**Trigger:** "+ Nouvel ordre de fabrication" → `setWizardOpen(true)` in `app/production/page.tsx`  
**Component:** `components/production/production-wizard.tsx` + `allocation-step.tsx`  
**Handler chain:**
1. Step 1: recipe select (Combobox) + nombre de broches (input, min 1)
2. Step 2: `buildFifoDefaults()` pre-fills FIFO allocations; `AllocationStep` shows per-ingredient rows with `computeShortfall()` real-time "manquant" amber badge; step blocked while shortfall > 0
3. Step 3: récapitulatif renders recipe, dates, broche DLC (computeBrocheDlc), lot list
4. `handleConfirm()`: `store.updateRawMaterial(id, {quantiteRestante: Math.round(... - used, 2dp)})` per lot; `generateLotNumber(new Date(), seq)` × N; `store.addProductionOrder(order)`; `store.addFinishedProduct(fp)` × N; `toast.success("Production confirmée — N broches (recipe)")` with "Voir la traçabilité" action calling `router.push(/tracabilite?lot=...)`

**Status: WIRED** — handleConfirm at lines 83–121 of production-wizard.tsx confirmed by grep

### Step 3 — Livrer 2 broches à un client

**Route:** `/livraisons`  
**Trigger:** "+ Nouvelle livraison" → `setDialogOpen(true)` in `app/livraisons/page.tsx`  
**Components:** `components/livraisons/new-delivery-dialog.tsx` → `deliveries-table.tsx`  
**Handler chain:**
1. `NewDeliveryDialog`: Client combobox (name→id), DatePicker (default today), broches checkbox list (getInStockBroches filters `statut === "en_stock"`, sorted by DLC ascending), Notes textarea
2. `onSubmit()`: `addDelivery({statut:"preparee", ...})` → `toast.success("Livraison préparée — N broche(s) pour clientName")` → `form.reset(); onOpenChange(false)`
3. Broches remain `en_stock` until explicit second action
4. `DeliveriesTable`: preparee rows show "Marquer comme livrée" button → `setPendingDelivery(row)` → AlertDialog "Confirmer la livraison" (irreversible warning)
5. `handleConfirmLivree()`: loops over `brochesLivrees` → `store.updateFinishedProduct(fpId, {statut:"livree", livraisonId})` → `store.updateDelivery(id, {statut:"livree"})` → `toast.success("Livraison confirmée — N broche(s) livrée(s) à clientName")`

**Status: WIRED** — both handler chains confirmed at lines 61–77 (deliveries-table.tsx) and 90–102 (new-delivery-dialog.tsx)

### Step 4 — Recherche traçabilité sur le n° de lot fournisseur

**Route:** `/tracabilite`  
**Component:** `app/tracabilite/page.tsx` → `components/tracabilite/tracabilite-upstream.tsx`  
**Handler chain:**
1. User types supplier lot number (or clicks "Voir un exemple : tracer un lot de matière première" chip)
2. `applyChip(lotNumber)` calls `setQuery(lotNumber)` + `triggerSearch(lotNumber)` + `router.replace(?lot=...)`
3. `triggerSearch(query)`: `detectLotType(trimmed)` — returns `"supplier"` for non-TK format → `findSupplierLot(trimmed, rawMaterials)` → if found: `setSearchState({type:"upstream", rm})`; if not: `setSearchState({type:"not-found"})`
4. Page renders `<TracabiliteUpstream rm={rm} />` — three TracabiliteSection cards:
   - Section 1 (Matière première): all lot info — fournisseur, certificat, dates, température via DlcBadge
   - Section 2 (Ordres de fabrication): `getProductionOrdersForRm(rm.id, productionOrders)` with quantiteUtilisee per order
   - Section 3 (Clients impactés): `getClientsImpactes(rm.id, productionOrders, finishedProducts, deliveries, customers)` — full 6-step chain, renders client nom + delivery date + internal lot number
5. Visual connector left-rail (lines 82–86 in tracabilite-upstream.tsx) connects the three sections

**Status: WIRED** — triggerSearch at lines 38–64 of page.tsx; detectLotType, findSupplierLot confirmed by grep; getProductionOrdersForRm and getClientsImpactes wired at lines 52–58 of tracabilite-upstream.tsx

### Step 5 — Exporter le dossier traçabilité en PDF

**Route:** `/tracabilite` (result view)  
**Component:** `components/tracabilite/tracabilite-upstream.tsx` (or downstream)  
**Handler chain:**
1. User clicks "Exporter dossier traçabilité (PDF)" button (`onClick={handlePrint}`)
2. `handlePrint` = `useReactToPrint({contentRef: printableRef})` — triggers browser print dialog
3. `printableRef` points to `<TracabilitePrintable ref={printableRef}>` which wraps all 3 TracabiliteSection cards with `className="print-target"`
4. `app/globals.css` @media print block (lines 59–72): `body * { visibility: hidden }` + `.print-target, .print-target * { visibility: visible }` + `.print-target { position: absolute; top: 0; left: 0; width: 100% }` — sidebar, header, and chrome are hidden; only traceability content prints

**Status: WIRED** — useReactToPrint at lines 47–50, onClick at line 72 of tracabilite-upstream.tsx; TracabilitePrintable className="print-target" at line 21; @media print at globals.css lines 59–72; all confirmed by grep
