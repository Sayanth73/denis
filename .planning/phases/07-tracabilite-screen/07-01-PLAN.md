Pure helpers for Phase 7. No React imports, no side effects, no Zustand imports. All parameters are explicit — no store access.

```typescript
/**
 * Pure helpers for the Traçabilité screen (Phase 7).
 * No React. No Zustand. All functions operate on domain types.
 *
 * Lot detection regex (locked — 07-CONTEXT.md):
 *   Internal broche: /^TK-\d{4}-\d{4}-\d{3}$/
 *   Supplier lot: anything else
 */

import type {
  RawMaterial,
  FinishedProduct,
  ProductionOrder,
  Customer,
  Delivery,
  Recipe,
} from "./types";
```

Implement these 6 functions exactly:

**`detectLotType(input: string): "broche" | "supplier" | null`**
- If `input.trim() === ""` → return `null`
- If `/^TK-\d{4}-\d{4}-\d{3}$/.test(input.trim())` → return `"broche"`
- Otherwise → return `"supplier"`

**`findSupplierLot(input: string, rawMaterials: RawMaterial[]): RawMaterial | null`**
- Return `rawMaterials.find(rm => rm.numeroLotFournisseur === input.trim()) ?? null`

**`findBroche(input: string, finishedProducts: FinishedProduct[]): FinishedProduct | null`**
- Return `finishedProducts.find(fp => fp.numeroLotInterne === input.trim()) ?? null`

**`getProductionOrdersForRm(rmId: string, productionOrders: ProductionOrder[]): { order: ProductionOrder; quantiteUtilisee: number }[]`**
- Iterate `productionOrders`; for each order, look in `order.matieresPremieresUtilisees` for an entry where `rawMaterialId === rmId`.
- If found: include `{ order, quantiteUtilisee: entry.quantiteUtilisee }`.
- Sort result by `order.date` descending (most recent first).

**`getClientsImpactes(rmId, productionOrders, finishedProducts, deliveries, customers): { customer: Customer; delivery: Delivery; broches: FinishedProduct[] }[]`**

Trace path (upstream to downstream):
1. Find all orders that used `rmId` (same as `getProductionOrdersForRm`).
2. For each such order, collect all `brochesProduites` IDs.
3. Resolve each ID in `finishedProducts` — keep only those with `statut === "livree"` and `livraisonId` set.
4. Group by `livraisonId` — for each unique delivery:
   a. Find the delivery in `deliveries`.
   b. Find the customer via `delivery.customerId` in `customers`.
   c. If both found: add `{ customer, delivery, broches: [all fps for this deliveryId] }`.
5. Sort by `delivery.date` descending.
6. Return `[]` safely if any intermediate lookup fails.

**`getRecipeForOrder(broche: FinishedProduct, productionOrders: ProductionOrder[], recipes: Recipe[]): { order: ProductionOrder; recipe: Recipe } | null`**
- Find `order = productionOrders.find(o => o.id === broche.productionOrderId)`.
- If not found: return `null`.
- Find `recipe = recipes.find(r => r.id === order.recipeId)`.
- If not found: return `null`.
- Return `{ order, recipe }`.

Constraints:
- No `: any` anywhere.
- No `TODO/FIXME/XXX`.
- File ≤ 150 lines.
- JSDoc comment on each exported function.

```typescript
type TracabiliteSectionProps = {

  heading,
```

```typescript
import * as React from "react";

type TracabilitePrintableProps = {
  children: React.ReactNode;
};

/**
 * Wraps the traceability result content in a div with className="print-target".
 * The print-target class is referenced by the @media print block in app/globals.css
 * to isolate the printable region (hides everything else on the page).
 *
 * Used by TracabiliteUpstream and TracabiliteDownstream via useReactToPrint contentRef.
 */
export const TracabilitePrintable = React.forwardRef<
  HTMLDivElement,
  TracabilitePrintableProps
>(function TracabilitePrintable({ children }, ref) {
  return (
    <div ref={ref} className="print-target">
      {children}
    </div>
  );
});

TracabilitePrintable.displayName = "TracabilitePrintable";
```
