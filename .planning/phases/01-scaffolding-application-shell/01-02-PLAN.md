4. **Export `getActiveLabel(pathname: string): string`** — returns the matching label or `"TraceKebab"` as a default fallback (defensive — should never trigger on the 6 known routes). Implementation:
   ```ts
   export function getActiveLabel(pathname: string): string {
     // Exact match for "/" (otherwise every path would match it as a prefix).
     if (pathname === "/") return NAV_LABELS["/"];
     // Prefix match for the others (e.g., a future /clients/123 detail route still shows "Clients").
     const match = NAV_ITEMS.find(
       (item) => item.route !== "/" && pathname.startsWith(item.route)
     );
     return match?.label ?? "TraceKebab";
   }
   ```

5. **Export `isActiveRoute(pathname: string, route: string): boolean`** — used by `<NavItem />` to decide whether to apply active-state styling. Same exact-vs-prefix split as `getActiveLabel`:
   ```ts
   export function isActiveRoute(pathname: string, route: string): boolean {
     if (route === "/") return pathname === "/";
     return pathname.startsWith(route);
   }
   ```

The whole file should be ~35 lines. No imports beyond TypeScript's built-in `Record`. Do NOT import lucide-react here — components resolve the icon by name themselves (Task 2).

Why a name-based icon resolution rather than holding the React component reference: keeps `lib/nav.ts` a pure data module, importable from server components without dragging the lucide-react bundle along, and decouples the data from the renderer.

Behavior:

- Compute `const active = isActiveRoute(usePathname(), route);` (import from `@/lib/nav`).
- Style per UI-SPEC §Color "Hover/focus/active interaction states" table:
  - Resting (inactive): `bg-transparent text-muted-foreground`
  - Hover: `bg-zinc-100 text-foreground`
  - Active route: `bg-zinc-100 text-foreground` PLUS a 2px-wide blue-600 left strip
  - Focus ring: `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`
- Layout per UI-SPEC §Spacing "Layout-specific dimensions": `h-9` (36px), `px-3` (12px horizontal), `gap-3` (12px icon→label), `rounded-md`, full-row clickable.
- The 2px blue-600 active strip: render as a sibling element absolutely positioned on the left edge inside the link, only when `active` is true. Example structure:

```tsx
<Link
  href={route}
  className={cn(
    "relative flex h-9 items-center gap-3 rounded-md px-3 text-sm font-medium transition-colors",
    "focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2",
    active
      ? "bg-zinc-100 text-foreground"
      : "text-muted-foreground hover:bg-zinc-100 hover:text-foreground",
  )}
>
  {active && (
    <span
      aria-hidden="true"
      className="absolute left-0 top-1.5 bottom-1.5 w-0.5 rounded-full bg-primary"
    />
  )}
  <Icon size={16} className="shrink-0" />
  <span>{label}</span>
</Link>
```
