# Phase 1 — UI Design Contract

> Visual and interaction contract for the application shell. This is the **canonical design system reference for phases 1–9** of TraceKebab. Subsequent phases inherit every token, scale, color, copy convention, and component pattern declared here. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Scope note:** Phase 1 only renders the shell (sidebar + header + six route stubs). However, because Phase 1 is the first UI phase, this UI-SPEC declares the full design system so Phases 2–9 can reference a single source of truth instead of redeclaring tokens. Tokens marked **[Phase 1 surface]** are exercised in this phase; tokens marked **[Inherited by phase N+]** are declared here but first used later. All tokens in this contract are locked unless a later phase explicitly amends with a CHANGELOG entry.

## Spacing Scale

All Tailwind spacing tokens used in TraceKebab map to multiples of 4px (Tailwind's default 4px base — `gap-1` = 4px, `gap-2` = 8px, `gap-4` = 16px, `gap-6` = 24px, `gap-8` = 32px, `gap-12` = 48px, `gap-16` = 64px). The team must pick from this scale; no arbitrary values like `gap-[13px]`.

| Token | Tailwind class | Value | Usage |
|-------|---------------|-------|-------|
| xs | `*-1` | 4px | Icon-to-label gap inside a button or sidebar item |
| sm | `*-2` | 8px | Inline label↔input spacing, table cell internal vertical padding |
| md | `*-4` | 16px | Default element spacing, card internal padding (`p-4`) on dense cards |
| lg | `*-6` | 24px | Section padding, card padding on KPI cards (`p-6`), main content `px-6` |
| xl | `*-8` | 32px | Vertical gap between major page sections (form ↔ table, KPI row ↔ alerts row) |
| 2xl | `*-12` | 48px | Page-top breathing room above search bar in `/tracabilite` (its hero treatment) |
| 3xl | `*-16` | 64px | Reserved for empty-state vertical centering blocks |

**Layout-specific dimensions** (Phase 1 surface):

| Element | Dimension | Tailwind |
|---------|-----------|----------|
| Sidebar width | 240px (fixed) | `w-60` |
| Header height | 56px (fixed) | `h-14` |
| Sidebar item height | 36px | `h-9` |
| Sidebar item horizontal padding | 12px | `px-3` |
| Sidebar item icon-to-label gap | 12px | `gap-3` |
| Sidebar internal vertical padding | 16px | `py-4` |
| Header horizontal padding | 24px | `px-6` |
| Main content area padding | 24px | `px-6 py-6` |
| Main content max width | none (fills viewport minus sidebar) | n/a |

**Table density** [Inherited by phase 3+, per PRD §7]:

| Element | Dimension | Tailwind |
|---------|-----------|----------|
| Table cell vertical padding | 8px | `py-2` |
| Table cell horizontal padding | 12px | `px-3` |
| Table row min height | 36px | `min-h-9` (effective) |
| Table header row vertical padding | 12px | `py-3` |

Exceptions: none. The `*-1` (4px) token is allowed as the smallest gap because the lucide icon adjacency in sidebar items reads better at 12px gap (`gap-3`) than 16px, and chip inner gaps need 4px granularity for badges.

## Typography

Four roles, two weights. No other sizes/weights may appear in the codebase; the auditor will fail dimension 4 if any text uses unlisted values.

| Role | Size | Weight | Line Height | Tailwind | Where used |
|------|------|--------|-------------|----------|------------|
| Body | 14px | 400 (regular) | 1.5 (21px) | `text-sm leading-6 font-normal` | Default body text, table cells, form inputs, sidebar inactive labels, descriptive copy |
| Label | 14px | 500 (medium) | 1.4 (~20px) | `text-sm font-medium leading-tight` | Form field labels, sidebar active label, table column headers, button text |
| Heading | 20px | 600 (semibold) | 1.3 (26px) | `text-xl font-semibold leading-snug` | Header page title, dialog titles, section headings inside cards |
| Display | 28px | 600 (semibold) | 1.2 (~34px) | `text-3xl font-semibold leading-tight` | KPI card numeric values, traçabilité hero search bar leading number |

Total: **4 sizes × 2 weights = 8 distinct text styles.** The dense 14px body is mandated by PRD §7 ("texte 14px") and is intentional — TraceKebab is a data-dense B2B tool.

**Monospace exception:** lot numbers (supplier and internal), DLC ISO dates, and numeric IDs render in `font-mono text-sm` (Geist Mono, 14px) so they are visually distinguishable as identifiers and align by character. Used in: table cells showing `numeroLotFournisseur` / `numeroLotInterne`, traçabilité section cards. Not used in Phase 1 surface.

## Copywriting Contract

All copy is **formal French B2B**, second-person plural ("vous"), neutral and action-oriented, no exclamation marks, no emojis, no anglicisms where a French equivalent exists. Punctuation: French typographic conventions (e.g., space before `?`, `:`, `!`, `;` is acceptable but not mandatory in UI labels — be consistent within a single sentence).

### Tone rules
- **Formal but not stiff.** "Réceptionnez votre premier lot" is correct (imperative, formal-vous). "Récupère ton lot" is wrong (informal-tu).
- **Action-oriented verbs.** Empty-state CTAs use imperative: "Réceptionnez", "Créez", "Recherchez". Never "Cliquez ici".
- **No marketing fluff.** Never "Excellente nouvelle !", "Prêt à commencer ?". The user is a meat transformer, not a consumer.
- **No emojis in copy strings** (sidebar lucide icons are components, not glyphs in a string).
- **Currency & numbers**: Swiss French — kg with non-breaking space (`25 kg`, not `25kg`); decimal separator is comma in numerical inputs/outputs; ISO dates in tables but `JJ.MM.AAAA` is acceptable for display [decided in phase 3].

## Density & Visual Rhythm

- **Information density:** PRD §7 says "tableaux denses". Tables use `text-sm py-2 px-3` cells. Cards use generous internal padding (`p-6`) to compensate — cards are the breathing room, tables are the dense surface.
- **Border radius:** shadcn default `rounded-md` (6px) for buttons, cards, dialogs, badges, table containers. No `rounded-full` except small dots in alert indicators. No `rounded-lg`/`rounded-xl` — would feel consumer-y.
- **Border weight:** `border` (1px) everywhere. No 2px borders except the active-sidebar-item left-strip.
- **Shadow:** shadcn default `shadow-sm` on cards; `shadow-md` on dialogs (popover layer); `shadow-none` on tables. No decorative shadows. No glow effects.
- **Animation:** shadcn defaults only — dialog fade+scale, popover fade, toast slide. No custom transitions in Phase 1. No hover-lift card effects.
