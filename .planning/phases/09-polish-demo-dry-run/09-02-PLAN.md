**Step 2 trace — Production wizard:**

- Store action `addProductionOrder` should also decrement `quantiteRestante` on each consumed raw material via `updateRawMaterial`

Confirm in `lib/store.ts` that `addProductionOrder` handles the side-effect of decrementing quantiteRestante. If it only adds the order without decrementing, that is a **blocker** — add the decrement logic inside `addProductionOrder`:
```typescript
addProductionOrder: (o) => set((s) => {
  // Decrement quantiteRestante for each consumed lot
  const updatedRMs = s.rawMaterials.map((rm) => {
    const used = o.matieresPremieresUtilisees.find((m) => m.rawMaterialId === rm.id);
    if (!used) return rm;
    return { ...rm, quantiteRestante: Math.max(0, rm.quantiteRestante - used.quantiteUtilisee) };
  });
  return {
    rawMaterials: updatedRMs,
    productionOrders: [...s.productionOrders, o],
    finishedProducts: [...s.finishedProducts, ...o.brochesProduites],
  };
}),
```
