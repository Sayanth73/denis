## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (gap-1), inline badge padding (px-1) |
| sm | 8px | Compact cell padding (py-2 px-3 = 8px/12px) |
| md | 16px | Default element spacing, dialog body padding |
| lg | 24px | Header row bottom margin (mb-6), card padding (p-5) |
| xl | 32px | Section vertical rhythm |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Search input wrapper: `mb-4` (16px) between header row and table — matches md token.
- Table header cells: `py-3 px-3` (12px/12px). Non-standard vertical but established in codebase — do not change.
- Table body cells: `py-2 px-3` (8px/12px). Established pattern — do not change.
- Icon-only action buttons: `size="sm"` ghost buttons (shadcn default 32px height) — no 44px touch-target exception needed (desktop-only app per OUT OF SCOPE).

Source: Measured from `components/clients/clients-table.tsx` and `app/clients/page.tsx` (established patterns).

## Interaction Contracts

### 1. Date format audit

- **Rule:** Every date shown to the user must render as `DD.MM.YYYY` (e.g., `05.05.2026`).
- **Helper:** `formatDate(iso: string)` from `lib/raw-materials.ts` — already correct.
- **DLC dates:** `DlcBadge` component uses its own `formatDateJjMmAaaa` — already correct.
- **Audit scope:** Scan all `.tsx` files for raw ISO string renders (patterns: `{d.date}`, `{rm.dateReception}`, `{f.dateFacture}`, `{f.datePaiement}`) not wrapped in `formatDate()`.
- **Dashboard activity:** `formatRelativeDate` on the activity column is a relative label ("aujourd'hui", "hier") — this is intentional, not an ISO date render. No change required.
- **Expected outcome:** Minimal or zero code changes. Confirm compliance by inspection.

### 2. Delete confirmation — matière première

Layout in `raw-materials-table.tsx`:
- Add an 8th column as the rightmost column, width `6%`, no header label.
- Column contains a ghost size-sm icon-only button with `Trash2` icon (size={14}).
- Button class: `variant="ghost" size="sm" type="button" aria-label="Supprimer" className="text-destructive hover:text-destructive"`.
- Button renders for ALL rows unconditionally — the deletion guard runs on click, not on render.

Deletion guard (runs on Trash2 click, before showing dialog):
1. Check if any `productionOrders` entry has `lotsUtilises` containing the raw material's `id`.
2. If YES (RM is referenced): call `toast.error("Impossible de supprimer un lot déjà utilisé en production.")`. Do NOT open AlertDialog. Set no state.
3. If NO (RM is unreferenced): set `pendingDeleteId` to the RM's `id`. AlertDialog opens.

AlertDialog copy (matière première):
- Title: `Supprimer la matière première`
- Description line 1: `Êtes-vous sûr de vouloir supprimer {rm.nom} ?`
- Description line 2: `Cette action est irréversible.`
- Cancel button: `Annuler` (default AlertDialogCancel)
- Confirm button: `Supprimer` — class `bg-destructive text-destructive-foreground hover:bg-destructive/90`

State: `pendingDeleteId: string | null` local to `RawMaterialsTable`.
On confirm: `useTraceabilityStore.getState().deleteRawMaterial(pendingDeleteId)` + `toast.success("Matière première supprimée — {nom}")`.

Pattern source: `components/clients/clients-table.tsx` — copy verbatim, adapt entity name.

### 3. Search inputs

Placement: `<div className="mb-4">` between the header row `<div className="flex items-center justify-between mb-6 h-9">` and the table/EmptyState.

Input spec:
- Component: `<Input>` from `@/components/ui/input`
- Width: `w-64` (256px)
- Placeholder: `"Rechercher..."`
- No icon prefix (plain Input, matches existing form inputs in app)
- `value={query}` controlled, `onChange={(e) => setQuery(e.target.value)}`

State: `const [query, setQuery] = React.useState("")` — local to the page component (not in table component, not in store).

Filter logic: `React.useMemo` computing filtered list, passed as prop to table component. Case-insensitive substring match via `.toLowerCase().includes(query.toLowerCase())`.

Show/hide: Search input renders even when list is empty (to allow typing before data loads). However when `!hasHydrated`, search input is NOT rendered (consistent with existing hydration guard pattern that only renders the disabled header skeleton).

**Per-page filter fields:**

| Page | File | Filter fields |
|------|------|--------------|
| /clients | `app/clients/page.tsx` | `customer.nom` |
| /livraisons | `app/livraisons/page.tsx` | Customer name (`customers.find(c => c.id === d.customerId)?.nom`) OR `d.date` (ISO substring — e.g. "2026-05" matches May deliveries) |
| /matieres-premieres | `app/matieres-premieres/page.tsx` | `rm.nom` OR `rm.fournisseur` |
