## Interaction Contracts

### 1. Tab header row

Layout: `<div className="flex items-center justify-between mb-4">` at the top of `RecettesTab`.

Left side: section heading or empty (no heading needed â€” tab label already provides context).

Right side: `<Button variant="default" size="sm" onClick={() => setCreateOpen(true)}>` with `<Plus size={14} className="mr-1.5" aria-hidden="true" />` and label `Nouvelle recette`.

### 2. Recipe card layout (updated)

Each recipe card: `<div className="rounded-md border bg-background p-4">`.

Card header row: `<div className="flex items-center justify-between">`.
- Left: recipe name `<span className="text-sm font-semibold text-foreground">{recipe.nom}</span>`.
- Right: action buttons group + price.

Action buttons group (right side of header row): `<div className="flex items-center gap-1">`.
- Edit2 button: `<Button variant="ghost" size="sm" type="button" aria-label="Modifier" onClick={() => setEditRecipe(recipe)}>` with `<Edit2 size={14} aria-hidden="true" />`.
- Trash2 button: `<Button variant="ghost" size="sm" type="button" aria-label="Supprimer" className="text-destructive hover:text-destructive" onClick={() => handleDeleteClick(recipe.id)}>` with `<Trash2 size={14} aria-hidden="true" />`.

Price display: `<span className="text-sm text-muted-foreground tabular-nums">{recipe.prixParDefautHT.toFixed(2)} CHF/kg HT</span>` â€” placed to the left of the action buttons group or in the left column; executor may place price on the left and actions on the right to avoid crowding.

Existing composition section below the divider remains unchanged (read-only display).

### 3. Create dialog ("Nouvelle recette")

Trigger: "Nouvelle recette" button in the tab header row.

State: `const [createOpen, setCreateOpen] = React.useState(false)`.

Dialog spec:
- `<Dialog open={createOpen} onOpenChange={setCreateOpen}>`.
- `<DialogContent className="sm:max-w-[480px]">`.
- Title: `Nouvelle recette`.
- Description: `Renseignez le nom et le prix de la nouvelle recette.`.

Form fields (vertical stack, `space-y-4`):
1. `nom` â€” `<Input type="text" placeholder="Nom de la recette" maxLength={120} />` â€” required, min 1 character.
2. `prixParDefautHT` â€” `<Input type="number" min="0" step="0.01" placeholder="0.00" />` â€” required, must be a positive number. Label: `Prix par dĂ©faut (CHF/kg HT)`.

Footer buttons (right-aligned):
- `<Button type="button" variant="outline" onClick={() => setCreateOpen(false)}>Annuler</Button>`.
- `<Button type="submit" variant="default" disabled={!isValid}>CrĂ©er</Button>` â€” disabled until `nom` is non-empty and `prixParDefautHT` is a positive number.

On submit: call `addRecipe({ id: crypto.randomUUID(), nom, prixParDefautHT, poidsTotal: 0, composition: [] })`. Close dialog. Show `toast.success("Recette crĂ©Ă©e â€” {nom}")`.

### 4. Edit dialog

Trigger: Edit2 icon button on a recipe card.

State: `const [editRecipe, setEditRecipe] = React.useState<Recipe | null>(null)`.

Dialog: opens when `editRecipe !== null`. `open={editRecipe !== null}` `onOpenChange={(open) => { if (!open) setEditRecipe(null); }}`.

Same layout as create dialog except:
- Title: `Modifier la recette`.
- Description: `Mettez Ă  jour le nom ou le prix de la recette.`.
- Pre-filled: `nom` and `prixParDefautHT` from `editRecipe`.
- Submit button label: `Enregistrer`.

On submit: call `updateRecipe(editRecipe.id, { nom, prixParDefautHT })`. Close dialog. Show `toast.success("Recette mise Ă  jour")`.

### 5. Delete flow

Trigger: Trash2 icon button on a recipe card.

Handler `handleDeleteClick(id: string)`:
1. Check if any `productionOrders` entry has `recipeId === id`.
2. **If referenced** (recipe used in a production order): call `toast.error("Impossible de supprimer : cette recette est utilisĂ©e dans des ordres de fabrication.")`. Do NOT open AlertDialog. Do NOT set `pendingDeleteId`.
3. **If unreferenced**: set `pendingDeleteId` to the recipe's `id`. AlertDialog opens.

State: `const [pendingDeleteId, setPendingDeleteId] = React.useState<string | null>(null)`.

AlertDialog spec:
- `open={pendingDeleteId !== null}` `onOpenChange={(open) => { if (!open) setPendingDeleteId(null); }}`.
- Title: `Supprimer la recette`.
- Description line 1: `ĂŠtes-vous sĂ»r de vouloir supprimer Â« {nom} Â» ?`.
- Description line 2: `Cette action est irrĂ©versible.`.
- Cancel button: `<AlertDialogCancel>Annuler</AlertDialogCancel>`.
- Confirm button: `<AlertDialogAction className="bg-destructive text-destructive-foreground hover:bg-destructive/90" onClick={handleConfirmDelete}>Supprimer</AlertDialogAction>`.

On confirm (`handleConfirmDelete`):
1. Cascade: `customers.filter(c => c.tarifs.some(t => t.recetteId === pendingDeleteId)).forEach(c => updateCustomer(c.id, { tarifs: c.tarifs.filter(t => t.recetteId !== pendingDeleteId) }))`.
2. Call `deleteRecipe(pendingDeleteId)`.
3. Show `toast.success("Recette supprimĂ©e")`.
4. Set `setPendingDeleteId(null)`.

Source: CONTEXT.md Â§Delete Behavior and Â§Established Patterns (clients-table).
