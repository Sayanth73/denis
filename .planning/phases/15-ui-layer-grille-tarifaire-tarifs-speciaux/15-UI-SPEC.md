## Typography

| Role | Size | Weight | Line Height | Class |
|------|------|--------|-------------|-------|
| Body / Label | 14px | 400 (regular) | 1.5 | `text-sm` |
| Section heading | 16px | 600 (semibold) | 1.2 | `text-base font-semibold` |
| Muted / secondary text | 14px | 400 (regular) | 1.5 | `text-sm text-muted-foreground` |

Notes:
- Only 2 font weights: 400 (body, labels, muted) and 600 (section headings). Label distinction is structural, not typographic.
- Section headings match existing pattern: `<h2 className="text-base font-semibold">` (from `app/parametres/page.tsx`) and `<h3 className="text-base font-semibold">` (from `app/clients/[id]/page.tsx`)
- Table header text: `text-sm text-muted-foreground` — drops `font-medium` to stay within 2-weight limit
- Numeric values (CHF/kg): `tabular-nums` on readonly cells for aligned display
- No Display size needed — this phase has no hero headings

Source: `app/parametres/page.tsx`, `app/factures/page.tsx`, `app/globals.css`

## Component Inventory

### Grille tarifaire section (Paramètres page)

**Location:** `app/parametres/page.tsx` — appended below the existing `<Form>` block, same `<div className="max-w-lg space-y-6">` container.

**Structure:**
```
<div className="space-y-4">
  <div>
    <h2 className="text-base font-semibold">Grille tarifaire</h2>
    <p className="text-sm text-muted-foreground mt-1">
      Prix par défaut au kg HT pour chaque recette. Utilisé lors de la
      génération automatique des factures.
    </p>
  </div>

  <div className="rounded-md border bg-background overflow-hidden">
    <table className="w-full">
      <thead className="bg-zinc-50">
        <tr>
          <th [text-sm font-medium text-muted-foreground py-3 px-3 border-b border-border text-left]>
            Recette
          </th>
          <th [... text-right]>
            Prix par défaut (CHF/kg HT)
          </th>
        </tr>
      </thead>
      <tbody>
        {recipes.map(recipe => (
          <tr [border-b border-border last:border-0]>
            <td [py-2 px-3 text-sm]>
              {recipe.nom}
            </td>
            <td [py-2 px-3 text-right]>
              <Input
                type="number"
                min={0}
                step={0.01}
                placeholder="25.00"
                className="w-24 text-right tabular-nums"
                value={localPrices[recipe.id] ?? ""}
              />
            </td>
          </tr>
        ))}
      </tbody>
    </table>
  </div>

  <Button type="button" onClick={handleSave}>
    Enregistrer
  </Button>
</div>
```

**State:** `localPrices: Record<string, string>` — useState keyed by recipe.id. Initialized from `recipe.prixParDefautHT` on mount (re-initialized when `hasHydrated` becomes true). String type allows clearing the input.

**Validation (inline, not form-level):**
- On save: each value must parse to `parseFloat` >= 0 and not NaN
- Error condition: show `<p className="text-sm text-destructive mt-1">Prix invalide pour {recipe.nom}. Saisissez un nombre ≥ 0.</p>` beneath the button
- Valid empty string treated as error (required field)

**Save action:**
1. Validate all 3 values — if any invalid, show inline error, abort
2. Call `updateRecipe(recipe.id, { prixParDefautHT: parseFloat(value) })` for each recipe
3. `toast.success("Grille tarifaire mise à jour")`

### Tarifs spéciaux section (Client detail page)

**Location:** `app/clients/[id]/page.tsx` — inserted between the client info card (`rounded-md border bg-background p-5 mb-6`) and the `<h3>Historique des livraisons</h3>` heading. Uses `mb-6` separator to maintain page rhythm.

**Structure:**
```
<div className="mb-6 space-y-4">
  <div>
    <h3 className="text-base font-semibold">Tarifs spéciaux</h3>
    <p className="text-sm text-muted-foreground mt-1">
      Laissez vide pour utiliser le prix par défaut de la recette.
    </p>
  </div>

  <div className="rounded-md border bg-background overflow-hidden">
    <table className="w-full">
      <thead className="bg-zinc-50">
        <tr>
          <th [text-sm font-medium text-muted-foreground py-3 px-3 border-b border-border text-left]>
            Recette
          </th>
          <th [... text-right]>
            Prix par défaut (CHF/kg)
          </th>
          <th [... text-right]>
            Override client (CHF/kg)
          </th>
        </tr>
      </thead>
      <tbody>
        {recipes.map(recipe => (
          <tr [border-b border-border last:border-0]>
            <td [py-2 px-3 text-sm]>
              {recipe.nom}
            </td>
            <td [py-2 px-3 text-sm text-right tabular-nums text-muted-foreground]>
              {recipe.prixParDefautHT.toFixed(2)} CHF/kg
            </td>
            <td [py-2 px-3 text-right]>
              <Input
                type="number"
                min={0}
                step={0.01}
                placeholder={recipe.prixParDefautHT.toFixed(2)}
                className="w-24 text-right tabular-nums"
                value={overrides[recipe.id] ?? ""}
              />
            </td>
          </tr>
        ))}
      </tbody>
    </table>
  </div>

  <Button type="button" onClick={handleSave}>
    Enregistrer
  </Button>
</div>
```

**State:** `overrides: Record<string, string>` — useState keyed by recipe.id. Initialize: for each recipe, `customer.tarifs.find(t => t.recetteId === recipe.id)?.prixHT?.toString() ?? ""`.

**Validation (inline, not form-level):**
- On save: each non-empty value must parse to `parseFloat` >= 0 and not NaN
- Empty string = no override = valid (removes entry from tarifs)
- Error condition: show `<p className="text-sm text-destructive mt-1">Override invalide pour {recipe.nom}. Saisissez un nombre ≥ 0 ou laissez vide.</p>` beneath the button

**Save action:**
1. Validate all non-empty values — if any invalid, show inline error, abort
2. Build `newTarifs`: for each recipe, if `overrides[id]` is non-empty and valid, include `{ recetteId: recipe.id, prixHT: parseFloat(overrides[id]) }`, otherwise exclude
3. Call `updateCustomer(customer.id, { tarifs: newTarifs })`
4. `toast.success("Tarifs mis à jour")`

**Edge case — clear override:**
- User empties a previously populated input → `overrides[id] = ""` → save excludes that entry from `newTarifs` → `customer.tarifs` no longer contains that recetteId → next invoice uses recipe default

## Interaction States

| State | Element | Visual |
|-------|---------|--------|
| Default | CHF/kg Input | Border: --border (#E4E4E7), bg: --background (#FFFFFF) |
| Focus | CHF/kg Input | Ring: 2px solid --ring (#2563EB), border-transparent |
| Error | Section (after save attempt) | `<p className="text-sm text-destructive">` below button |
| Saving | Button | No loading state — store writes are synchronous |
| Saved | Toast (sonner) | `toast.success(...)` — bottom-right, auto-dismiss 4s |
| Empty override | Input | Placeholder shows recipe default (e.g., "25.00") |
| Readonly default price | Cell | `text-muted-foreground tabular-nums` — not a form field |

No skeleton loaders in these sections — both pages already gate on `hasHydrated` before rendering.

## Layout Integration

### Paramètres page (`app/parametres/page.tsx`)

Existing outer container: `<div className="max-w-lg space-y-6">`.

The Grille tarifaire section appends inside this same container as a sibling after the `<Form>`. No wrapper change needed. The section naturally inherits the `max-w-lg` constraint, keeping the two-column table proportional at ~512px max width.

### Client detail page (`app/clients/[id]/page.tsx`)

No outer width constraint on this page (full column width). The Tarifs spéciaux table renders at full column width. Insert the section between:
- End of: `<div className="rounded-md border bg-background p-5 mb-6">` (client info card)
- Start of: `<h3 className="text-base font-semibold mb-4">Historique des livraisons</h3>`

The section carries its own `mb-6` to separate from the history heading below.
