**Save handler** — add after the useEffect:
```typescript
function handleSaveTarifs() {
  if (!customer) return;
  setTarifsError(null);
  for (const r of recipes) {
    const val = overrides[r.id] ?? "";
    if (val === "") continue; // empty = valid, means "use recipe default"
    if (isNaN(Number(val)) || Number(val) < 0) {
      setTarifsError(
        `Override invalide pour ${r.nom}. Saisissez un nombre ≥ 0 ou laissez vide.`,
      );
      return;
    }
  }
  const newTarifs: { recetteId: string; prixHT: number }[] = recipes
    .filter((r) => (overrides[r.id] ?? "") !== "")
    .map((r) => ({ recetteId: r.id, prixHT: Number(overrides[r.id]) }));
  const { updateCustomer } = useTraceabilityStore.getState();
  updateCustomer(customer.id, { tarifs: newTarifs });
  toast.success("Tarifs mis à jour");
}
```

Key edge case — clear override: when a user empties a previously populated input field and saves, `overrides[r.id]` is `""`. The `.filter` excludes that recipe from `newTarifs`, so the entry is absent from `customer.tarifs` after the write. The next invoice for this client will fall back to `recipe.prixParDefautHT` (per the Phase 14 `buildFacture` logic).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → store | CHF/kg values typed into number inputs flow into Zustand (localStorage-backed) on save |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | handleSaveGrille / handleSaveTarifs | mitigate | `Number(val)` used (not parseFloat) — rejects "25abc" as NaN; `val === ""` checked first for required fields; `Number(val) < 0` guard rejects negative prices before store write |
| T-15-02 | Tampering | handleSaveGrille | mitigate | Empty string is invalid for Grille tarifaire (recipe prices are required); explicit `val === ""` check aborts save with inline error |
| T-15-03 | Tampering | handleSaveTarifs | accept | Empty string is intentionally valid for Tarifs spéciaux (means "remove override"); this is correct business logic, not a vulnerability — it restores the recipe default |
| T-15-04 | Denial of Service | localStorage via Zustand persist | accept | Single-user local app; no multi-tenant surface; localStorage is bounded by browser limits (~5MB); 3 recipes × 1 number each is negligible |
| T-15-05 | Information Disclosure | readonly price cells | accept | Default prices are operator-visible business data, not PII or secrets; rendering them in read-only cells is correct and intentional |
</threat_model>
